# Changelog

全てのリリースや大きな変更はこのファイルに記録してください。上から新しい変更履歴を追加します。

## [0.1.0] - 2026-01-31
### Added
- 在庫一覧のモーダルから在庫数を +/- で調整して即時確定できる「在庫操作」体験を追加。
- `/api/inventory/issues` および `/recent-transactions` を通じて履歴を即時更新する API を追加し、フロントでの再読み込みなしに変更差分を反映。

### Changed
- 左ペインの用途ツリーと中央テーブルのフィルタの並び順や未設定表示を整理し、棚番ソートも安定化。
- モーダルの「確定」で `/inventory/inline-adjust` を使ってトランザクション管理し、`tx_type=ADJUST` のみ／マイナス在庫は許容せずに `reason` 省略可で在庫を反映する仕様が入った。
- Web UI の在庫行クリック時に棚番・メーカー・在庫・発注点・履歴情報を表示するよう強化。既存の出庫・調整 API はそのまま維持し、画面からの操作で後方互換を保つ。
- モーダル完結型にし、旧来の「持出し記録」フォームは撤去。

### Fixed
- `ZoneInfo("Asia/Tokyo")` でタイムスタンプを統一し、履歴・通知が常に JST で表示されるように整備。 

### Deps
- `tzdata` と `python-multipart` を `requirements.txt` に追加し、`zoneinfo`／FastAPI のフォーム処理が安定するように補完。
